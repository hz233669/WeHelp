第七屆 WeHelp 訓練營申請

1. 個人簡介和申請動機

我叫劉柏宏，畢業於大同大學生物工程學系，曾在生技公司擔任產品專員。

在工作過程中，我發現到生物產業的專業跨度相當大，讓我意識到自己更嚮往一個能持續累積專業的職涯，因此開始思考發展第二專長的可能，在探索過程中，我接觸到了網頁開發，進而了解到前端工程師是可以透過累積專案作品，來進行轉職，隨著學習的深入，也做出了一些專案作品，並體會到了網頁開發所帶給我的成就感，這讓我更加確定要往前端工程師發展。

因此，我決定報名 WeHelp 訓練營，希望能透過系統化、扎實的訓練，成為一位專業的軟體工程師。

2. 曾經做過哪些軟體工程技術相關的學習？若有作品請分享

我先後接觸了HTML、Bootstrap、JavaScript 與 Vue，並實際運用 Vue 框架完成了一個電商網站專案。目前有在學習資料結構與演算法的課程。

3. 如果參與這個訓練，會怎麼安排學習時間？

如果有幸能進入到 WeHelp 訓練營參與訓練，我會全職學習，將訓練營視作正式工作來對待，每天至少投入8小時進行學習與實作，並確實提交每日進度報告。盡全力完成每週任務和達成每一階段的目標。

4. 軟體技術日新月異，如何確定投入的領域是正確有回報的？

在目前還未轉職時，我認為可以先專注在程式語言的基礎功，前端部分專注於 JavaScript 和資料處理的能力為主。同時也需要加強資料結構與演算法等通用知識，因為這些基礎知識是比較不會隨著框架的更迭而被取代。

在轉職並累積一些工作經驗後，除了精進工作中的技術外，也需要觀察產業的趨勢，了解最近有什麼新技術或概念，可以選擇和自己工作有關的技術在工作之餘進行摸索。

此外也可以透過定期的面試，檢視自己的專業是否符合市場需求。

5. 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

幾年前家人因癌症住院，在治療過程中家人就有表達過不希望接受侵入性治療。直到接到病危通知的那天，雖然醫生盡力搶求，但情況還是沒有好轉。最後我選擇尊重家人的意願，簽下了不同意搶救的文件。

事後我常常反思思，懷疑自己是不是做錯決定，會不會再多爭取一點時間，家人就有機會好轉。這種矛盾讓我陷入深深的自責。為了調適情緒，我開始自我對話，提醒自己這不是放棄，而是尊重家人的選擇，讓他能減少不必要的痛苦。也讓我理解到在做重大決定，不被情緒左右，並遵從當事人的選擇，才是最正確的做法。

6. 關於這份申請網頁，分享一個開發時的技術心得？

在 CSS 中統一使用 rem 作為字體大小單位，這樣可以讓整個網頁在調整字體時保持一致比例，同時也更方便做RWD跟後續修改。

7. 如何看待人工智慧的發展對軟體工程師的影響?

人工智慧的發展改變了軟體工程師的工作方式。過去工程師的價值主要體現在專業技術的實作，但在如今AI工具的輔助下，如果仍停留在單純的技術輸出，將越來越容易被取代。

不過，工程師的職涯本來就是要不斷的學習。我認為人工智慧的發展只是加速了工程師的職涯過程，以前Junior工程師可能只需要先專注於技術研究，現在則需要更早開始理解產品架構，培養產品思維。將學習的重心從純粹的技術鑽研，轉移一部分到AI難以取代的軟實力部分，才能在未來的職涯上展現出自己的價值。
